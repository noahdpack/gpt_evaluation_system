# Contract Analyzer Test Knowledge Base

This mini knowledge base is designed for testing a contract analyzer GPT and a contract reviewer evaluation system.

It includes:
- Sample contracts with deliberately different risk profiles
- Internal-style legal review policies
- Clause playbooks for acceptable fallback positions
- Evaluation prompts and expected answer notes
- Retrieval traps, including similar-but-not-identical clause names, versioned policy guidance, and conflicting governing law preferences

Suggested test dimensions:
1. Can the GPT retrieve the correct policy file?
2. Can it distinguish sample contract facts from internal playbook standards?
3. Can it cite or reference knowledge base material rather than guessing?
4. Can it detect missing clauses and risky terms?
5. Can it reconcile older versus newer guidance?
